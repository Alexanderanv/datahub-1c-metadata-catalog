# Основные понятия

Расширение создаёт в DataHub два связанных представления одной и той же
конфигурации 1С: логический слой метаданных 1С и физический слой таблиц базы
данных.

## Логический слой 1С

Объекты метаданных 1С загружаются в DataHub как datasets:

- константы;
- справочники;
- документы;
- планы видов характеристик;
- планы счетов;
- планы видов расчёта;
- регистры сведений, накопления, бухгалтерии и расчёта;
- перечисления;
- табличные части.

В дереве навигации DataHub есть уровень информационной базы:

```text
1C:Enterprise
└── <отображаемое имя информационной базы>
    ├── Documents
    ├── Catalogs
    └── ...
```

Технический идентификатор информационной базы задаётся параметром
`infobase.name`. Имя, которое видно пользователю в UI, задаётся параметром
`infobase.display_name`.

## Физический слой базы данных

Если включён `ingestion.db_mapping=true`, коннектор дополнительно создаёт
datasets физических таблиц базы данных на основе API 1С `/db-mapping`. В именах
полей используется нейтральный префикс `db*`, потому что 1С может работать не
только с PostgreSQL, но и с другими СУБД.

Если физическая база данных работает на PostgreSQL, после коннектора 1С можно
запустить стандартный PostgreSQL source DataHub. Он заполнит реальные типы
колонок для тех же URN физических таблиц.

## Связи и lineage

Расширение разделяет три вида связей:

- DB mapping: в каких физических таблицах и колонках хранится объект 1С.
- Доменные связи: ссылочные связи 1С, например "объект ссылается на объект".
- Lineage: поток данных, например проведение документа в регистры.

Проведение документа представлено через process node DataHub. Это ближе к
реальной модели 1С: документ и его табличные части являются входом процесса
проведения, а процесс записывает данные в регистры.

## Кастомные аспекты

Кастомные аспекты - это типизированные группы свойств DataHub, добавленные этим
расширением. В них хранятся свойства, специфичные для 1С: вид объекта, DB
mapping и доменные связи. Перед записью этих аспектов DataHub GMS должен
загрузить кастомную метамодель.
