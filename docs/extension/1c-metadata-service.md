# Сервис метаданных 1С

Коннектор читает метаданные 1С через HTTP-сервис, развёрнутый в расширении 1С.
Сервис отдаёт только метаданные основной конфигурации. Объекты, реквизиты,
табличные части и сервисы, добавленные расширениями, намеренно скрываются:
стабильные UUID берутся из `ConfigDumpInfo.xml` основной конфигурации, и для
метаданных расширений в этом файле нет идентификаторов.

OpenAPI-контракт находится в:

```text
1c-metadata-service/api-spec/openapi.yaml
```

EDT-проект расширения находится в:

```text
1c-metadata-service/metadataservice-1c-extension
```

## Базовый URL

Ожидаемый URL сервиса выглядит так:

```text
http://1c-host/1c-test/hs/metadataservice
```

Укажите этот URL в recipe как `source.config.base_url` или через переменную
`ONEC_BASE_URL`.

## Нужные эндпойнты

Коннектор использует:

- `/health`;
- `/objects`;
- `/objects/{type}/{name}`;
- `/objects/{type}/{name}/tabular-parts`;
- `/db-mapping/{type}/{name}`;
- `/references`;
- `/lineage`;
- `/integration-services`.

Имена типов объектов в путях и query-параметрах используют английские имена
платформы 1С: `Catalogs`, `Documents`, `AccumulationRegisters` и т. д.
Отображаемые имена в DataHub при этом могут оставаться русскими.

## Развёртывание

Сервис разворачивается в той информационной базе 1С, из которой нужно читать
метаданные. Установка расширения в информационную базу осуществляется вручную
или CD процессами - на усмотрение пользователя расширения.
