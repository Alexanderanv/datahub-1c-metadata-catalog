# Проверка результата

После загрузки с нуля проверьте:

1. Платформа `1c-enterprise` существует.
2. Навигация содержит путь `1C:Enterprise -> <информационная база> -> папки
   типов объектов`.
3. Выбранные объекты 1С и табличные части присутствуют.
4. Стандартная вкладка `Columns` содержит поля 1С и описания там, где API 1С
   отдаёт синонимы или комментарии.
5. Вкладки `Properties 1C` и `Column mapping 1C` видны, если включены
   кастомные аспекты и установлена кастомная метамодель.
6. Физические DB datasets существуют, если включён `db_mapping=true`.
7. Реальные типы колонок физической базы появились после опционального прохода
   по базе данных.
8. Lineage показывает проведение документов и другие включённые потоки данных.
9. Доменные связи присутствуют как типизированные relationships, а не как
   стандартный lineage.
10. Опциональный MFE открывается и показывает тот же набор информационных баз и
    объектов.

Если UI DataHub обновляется с задержкой сразу после загрузки, сначала проверьте
состояние GMS, очередей ingestion и поискового индекса DataHub.
