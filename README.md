# Расширение DataHub для 1С:Предприятия

Репозиторий содержит расширение DataHub для загрузки и отображения метаданных
конфигураций 1С:Предприятия.

Проект оформлен как расширение к существующей установке DataHub, а не как
готовый промышленный дистрибутив DataHub. Эксплуатационные вопросы остаются
ответственностью владельца DataHub: TLS, SSO, резервное копирование,
мониторинг, подбор ресурсов, хранение секретов, отказоустойчивость и
обновления.

## Что входит в репозиторий

- Source-коннектор DataHub для метаданных 1С: `datahub/`.
- Кастомная метамодель DataHub для свойств и связей 1С: `custom-models/`.
- Контракт сервиса метаданных 1С и исходники EDT-расширения:
  `1c-metadata-service/`.
- Инструменты для работы с `ConfigDumpInfo.xml`, который даёт стабильные UUID
  объектов 1С:
  `1c-metadata-service/configdump-exporter/`.
- Опциональный DataHub Micro Frontend для просмотра данных 1С:
  `onec-metadata-explorer-mfe/`.
- Демонстрационный контур для пилотных проверок:
  `deploy/reference/`.
- Переносимые примеры recipe для загрузки: `examples/recipes/`.
- Корневой `Makefile` с переносимыми командами установки, сборки, тестов и
  демонстрационного запуска.

## Быстрый старт для локальной демонстрации

Для первого знакомства с проектом можно использовать локальный DataHub
quickstart:

```bash
cp datahub/.env.example .env
# заполните .env
# В демонстрационном recipe examples/recipes/1c-full.dhub.yaml
# не задан фильтр объектов и будут загружены все объекты конфигурации (а это долго)
# желательно создать свой recipe, скопировав демонстрационный
# и сузив object_filters.include_objects до необходимого, а затем указать путь в .env:
# ONEC_RECIPE=/path/to/1c-demo.dhub.yaml
make install
make demo.prepare
make demo.ingest
```

`make demo.ingest` по умолчанию использует
`examples/recipes/1c-full.dhub.yaml`. В этом примере пустые списки
`object_filters.include_objects` означают "загрузить все объекты выбранных
типов"; то же правило применяется к пустым спискам сервисов в
`integration_services.include_services`. Для реальной конфигурации 1С это
может быть долго и избыточно. Для первого пилота обычно лучше сделать свой
recipe на основе примера и оставить в нём только нужные справочники,
документы, регистры и сервисы.

Этот режим предназначен для демонстрации и пилотной проверки. Production
DataHub разворачивается и сопровождается отдельно.
