# Примеры recipe

Эти примеры можно использовать как переносимую стартовую точку для расширения
DataHub для 1С.

## Файлы

- `1c-minimal.dhub.yaml`: только логические метаданные 1С. Физический слой базы
  данных не создаётся.
- `1c-full.dhub.yaml`: логические метаданные 1С, кастомные аспекты, доменные
  связи, lineage и слой DB mapping.
- `db-postgres.dhub.yaml`: опциональный второй проход стандартным PostgreSQL
  source DataHub. Он заполняет реальные схемы физических таблиц на тех же DB
  URN, которые коннектор 1С создал на основе `/db-mapping`.

Для больших конфигураций 1С перед запуском в общем DataHub замените wildcard
списки явными именами объектов.

Если нужно, чтобы объекты, пропавшие из текущего набора загрузки,
автоматически помечались удалёнными в DataHub, включите `stateful_ingestion` и
задайте стабильный `pipeline_name`. Пример блока приведён в
`1c-full.dhub.yaml`.
